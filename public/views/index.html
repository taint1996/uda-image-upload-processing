Image Processor

Overview

I will be building an API that can be used in two different ways

As a simple placeholder API, the first API http://localhost:8080 allows you go to my index API. It will show an interview of project UI

The second API http://localhost:8080/api/images - It will show a list of images which is inside of our project from folder public/images

API List
# Endpoint Method Param Return Example Description
1 http://localhost:8080 GET None Go to Overview Image Processing None Go to Overview of Image Processing UI
2 http://localhost:8080/api/img/:img GET s - Size of image with 4 options: medium, small, tiny
w - Width of new image(in pixel)
h - Height of new image(in pixel) 		New version of image
  1. http://localhost:8080/api/img/fjord.jpg?s=full
  2. http://localhost:8080/api/img/encenadaport.jpg?s=medium
  3. http://localhost:8080/api/img/icelandwaterfall.jpg?s=small
  4. http://localhost:8080/api/img/palmtunnel.jpg?w=200&h=300
  5. http://localhost:8080/api/img/santamonica.jpg?w=40&h=40
 This API will return an image with the size according to the given Size (s) or according to the input width (w) and height (h).

This API prioritizes the given Size. If there are all 3 inputs s, w, h, it will give priority to return the image with the first input. into s.
